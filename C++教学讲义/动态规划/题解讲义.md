

## P2347 砝码称重

### 一、题目分析

**问题本质**：这是一个**多重背包**的变形问题。给定6种砝码，每种砝码有数量限制，问能称出多少种不同的重量。

**关键特点**：

-   每种砝码最多用 `a[i]` 个（数量受限）
-   总重量上限为1000
-   不需要输出具体方案，只问"能否凑出"

----------

### 二、状态定义

```cpp
dp[i][j]  // 布尔值：前i种砝码能否凑出重量j
```

维度|含义
---|---
`i`|考虑前 `i` 种砝码（1~6）
`j`|目标重量（0~1000）
值|`1` 表示可以凑出，`0` 表示不能

----------

### 三、状态转移方程

对于第 `i` 种砝码（重量为 `w[i]`，数量为 `a[i]`），枚举使用 `k` 个：

```text
dp[i][j] = OR{ dp[i-1][j - k*w[i]] }  
其中 k ∈ [0, a[i]] 且 k*w[i] ≤ j
```

**解释**：要凑出重量 `j`，可以选择：

-   用0个第 `i` 种砝码 → 依赖 `dp[i-1][j]`
-   用1个第 `i` 种砝码 → 依赖 `dp[i-1][j-w[i]]`
-   ...
-   用 `k` 个第 `i` 种砝码 → 依赖 `dp[i-1][j-k*w[i]]`

只要其中任意一种情况可行，`dp[i][j]` 就为真。

----------

### 四、初始化与边界

```cpp
// 重量为0永远可以凑出（一个砝码都不用）
for(int i = 0; i <= N; i++) 
    dp[i][0] = 1;
```

**为什么 `dp[i][0] = 1`？**

-   凑出重量0的方法：一个砝码都不用
-   这是所有状态的起点

----------

### 五、完整代码详解

```cpp
#include<iostream>
using namespace std;

const int N = 6, J = 1000;
int dp[N+3][1005];      // dp[i][j]：前i种砝码能否凑出重量j
int a[N+3];             // a[i]：第i种砝码的数量
int w[] = {0, 1, 2, 3, 5, 10, 20};  // 砝码重量，w[0]占位

void solve() {
    // 三重循环：物品种类 × 目标重量 × 使用个数
    for(int i = 1; i <= N; i++)           // 枚举第i种砝码
        for(int j = 1; j <= J; j++)       // 枚举目标重量j
            for(int k = 0; k <= a[i] && k*w[i] <= j; k++)  // 枚举使用k个
                dp[i][j] = dp[i][j] || dp[i-1][j - k*w[i]];
}

int main() {
    // 读入每种砝码的数量
    for(int i = 1; i <= N; i++) 
        cin >> a[i];
  
    // 初始化：重量0总是可达
    for(int i = 0; i <= N; i++) 
        dp[i][0] = 1;
  
    solve();
  
    // 统计答案：统计dp[6][1]到dp[6][1000]中有多少个1
    int ans = 0;
    for(int j = 1; j <= 1000; j++) 
        ans += dp[N][j];
  
    cout << "Total=" << ans << endl;
    return 0;
}
```

----------

### 六、复杂度分析

维度|复杂度|说明
---|---|---
**时间**|O(N × J × max(a[i]))|6 × 1000 × 最多数量
**空间**|O(N × J)|约 6 × 1001 ≈ 6KB

实际运行完全在限制范围内。

----------

### 七、易错点总结

易错点|正确做法
---|---
忘记初始化 `dp[i][0]`|所有 `dp[i][0]` 都要设为1
统计答案时包含重量0|题目明确要求"不包括一个砝码也不用"，所以从 `j=1` 开始统计
`k` 的循环条件|既要 `k <= a[i]`，也要 `k*w[i] <= j`，防止数组越界
砝码重量数组|`w[0]` 是占位符，实际从 `w[1]` 开始用

----------

### 八、思路总结

```text
问题转化：多重背包 → 判断可行性
核心思想：逐种考虑砝码，记录所有可达重量
状态设计：dp[i][j] 表示前i种能否凑出j
转移方式：枚举当前砝码使用个数，或运算合并状态
```

这是一道经典的**动态规划入门题**，掌握它有助于理解更复杂的多重背包问题。

## T677613 暗黑破坏神
 

### 题目核心分析 
这是一道**完全背包变式题**，核心特征： 
1. 有 `n` 个魔法（物品），`m` 个金币（背包容量）； 
2. 每个魔法可升级多次（完全背包：物品可重复选）； 
3. 魔法不同等级对应**不同效果值**（价值随选择次数变化）； 
4. 题目要求：输出**最大效果值** 和 **每个魔法的等级**，多解时输出**花费金币最少**的方案。 

**所以我们分两步来完成，首先使用完全背包求出最大效果值，再想办法记录花费金币最少的方案**

--- 

### 步骤一：纯完全背包求解「最大效果值」 
#### 模型转化 
- 背包容量：金币总数 `m` 
- 物品：每个魔法（可重复升级） 
- 物品花费：魔法每级价格 `c[i]` 
- 物品价值：不同等级的效果值 `w[i][k]` 
- 目标：不超过容量，总价值（效果值）最大 

#### 状态定义
 `dp[j]`：花费 `j` 金币能获得的**最大效果值** 
 
#### 状态转移 
 逆序枚举金币（完全背包标准写法），枚举每个魔法的所有等级，更新最大效果： 
 ```cpp 
 dp[j] = max(dp[j], dp[j-k*c[i]]+w[i][k]); 
 ``` 
 
 #### 求「最大效果值」完整代码
 
```cpp 
# include<iostream>
# include<cstring>
using namespace std;

int dp[505];
// c[]-每种物品价格，p[]-记录每种物品拥有的等级数，w[][]-每个魔法不同等级拥有的价值
int c[105], p[105], w[105][55];

int solve(int n, int m){
    for(int i=1;i<=n;i++)
        for(int j=m;j>=1;j--)
            for(int k=0;k<=p[i]&&j>=k*c[i];k++){   // j>=k*c[i]的判断条件一定不能少
                dp[j] = max(dp[j], dp[j-k*c[i]]+w[i][k]);
            }
    return dp[m];
}

int main(){
    int n, m, t;
    cin >> n >> m;
    for(int i=1;i<=n;i++){
        cin >> c[i] >> p[i];
        for(int k=1;k<=p[i];k++)
            cin >> w[i][k];
    }
    cout << solve(n, m) << endl;
    return 0;
}

``` 

  #### 代码说明 
  - 只求解**最大效果值**，无路径记录、无最小花费判断； 
  - 适用于只需要答案数值，不需要具体方案的场景。 
  
  --- 
  
  ### 步骤二：增加「最小花费」+「路径回溯」输出完整方案 
  #### 升级需求 
  1. 多解时，选择**花费金币最少**的方案； 
  2. 记录每个魔法的等级，**回溯输出方案**。 
  
  #### 新增状态定义 
  1. `min_cost[j]`：获得 `dp[j]` 最大效果时，**实际花费的最小金币数**； 
  2. `num[i][j]`：处理第 `i` 个魔法、剩余 `j` 金币时，该魔法学习的等级（用于回溯）。 

#### 升级转移规则（两个关键字条件）
```cpp 
/* 
效果值更大时，直接使用对应方案
效果值相同时，比较花费
*/

// i物品装k个价值更大
if(dp[j-k*c[i]]+w[i][k]>dp[j]) {
    num[i][j] = k;
    min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
}
// i物品装k个的价值和上一次擂台的价值相同，则比较花费
else if(dp[j-k*c[i]]+w[i][k]==dp[j]){
    // min_cost[j] = min(min_cost[j-k*c[i]] + k*c[i], min_cost[j]);
    if(min_cost[j-k*c[i]] + k*c[i] < min_cost[j]) {
        num[i][j] = k;
        min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
    }
}


``` 
#### 路径回溯逻辑 
1. 从后往前遍历所有魔法； 
2. 通过 `num[i][j]` 拿到当前魔法的等级； 
3. 扣除对应花费，继续回溯上一个魔法； 
4. 最终输出所有魔法的等级。

 #### 完整代码
```cpp 
// 多重背包变体
// 每个技能是一种物品，可以对同一个技能连续升级，所以是多重背包
// 技能的不同等级拥有不同的效果值，所以可以看作是价值会变化的多重背包
// 因为每种物品的价值会随装的数量而变化，所以只能使用枚举每种物品的所有等级打擂台的方式
// 金币数量即为背包容量

# include<iostream>
# include<cstring>
using namespace std;

int dp[505];
// c[]-每种物品价格，p[]-记录每种物品拥有的等级数，w[][]-每个魔法不同等级拥有的价值
int c[105], p[105], w[105][55];

// 此外，本题还要求记录每个物品的选取方案、花费的金币，总价值相同时，使用花费金币最少的情况，贪心
int num[105][505];  // 记录第i个物品，在总空间为j时，应使用的数量
int min_cost[505];  // 对应dp数组，记录每个最大总价所需的最小花费

int solve(int n, int m){
    for(int i=1;i<=n;i++)
        for(int j=m;j>=1;j--)
            for(int k=0;k<=p[i]&&j>=k*c[i];k++){   // j>=k*c[i]的判断条件一定不能少

                // i物品装k个价值更大
                if(dp[j-k*c[i]]+w[i][k]>dp[j]) {
                    num[i][j] = k;
                    min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
                }
                // i物品装k个的价值和上一次擂台的价值相同，则比较花费
                else if(dp[j-k*c[i]]+w[i][k]==dp[j]){
                    // min_cost[j] = min(min_cost[j-k*c[i]] + k*c[i], min_cost[j]);
                    if(min_cost[j-k*c[i]] + k*c[i] < min_cost[j]) {
                        num[i][j] = k;
                        min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
                    }
                }
                
                dp[j] = max(dp[j], dp[j-k*c[i]]+w[i][k]);
                
            }
    return dp[m];
}

int main(){
    int n, m, t;
    cin >> n >> m;
    for(int i=1;i<=n;i++){
        cin >> c[i] >> p[i];
        for(int k=1;k<=p[i];k++)
            cin >> w[i][k];
    }
    cout << solve(n, m) << endl;

    int res[105];  // 存储最终方案每个魔法的等级
    // 回溯找到最大价值下每种物品装入的数量
    int now = m;
    for (int i = n; i >= 1; i--) {
        res[i] = num[i][now];
        now -= res[i] * c[i];  // 减去当前魔法的花费
    }
    
    // 按顺序输出
    for (int i = 1; i <= n; i++) {
        cout << res[i] << endl;
    }

    return 0;
}

``` 

#### 代码说明 
1. **双关键字优化**：优先效果最大，相同效果选花费最少； 
2. **路径记录**：`num` 数组完整保存每一步选择，回溯即可得到方案； 
3. **边界处理**：数组初始化保证只有合法状态可以转移，无错误解。 

 --- 
 
### 总结 
1. **基础版**：纯完全背包，仅求解最大效果值； 
2. **进阶版**：增加最小花费判断 + 路径回溯，满足题目全部输出要求； 
3. 核心思想：**背包模板 + 双关键字DP + 路径回溯**，是背包问题输出方案的通用解法。

## P1064 [NOIP 2006 提高组] 金明的预算方案

> 此题不仅考察背包DP的应变能力，还考察代码实现基本功、问题分析能力

#### 整理题意：
总容量n，物品数m，物品体积为价格v[j]，最大化目标为v[j]*w[j]

**根据题目场景可得到的隐含信息**：每个物品最多只能买1个（0/1背包）。

**特殊点**：每个物品有主、附属性，一个主件最多有两个附件。

首先容易想到采用背包dp的思路，枚举前```i```个物品，枚举所有容量，计算每个状态时，枚举```i```物体的两种选择方式（1. 不装```i```，2. 装```i```同时连同对应的主件一起装），但很快会发现，将```i```物品连同主件一起装无法用我们通用的状态转移方程来表示，因为背包dp中状态```dp[i][j]```表示的是【前```i```个物品】,而这里```i```物品与他的主件并不是两个连续的物品，状态无法转移，所以行不通。**(可以尝试写一下)**

不妨换一个思路，**每个主件包含的附件数量是有限的**【0、1、2】，所以如果我们**对主件和背包容量建立状态```dp[i][j]```**，则状态可以连续转移，对于前```i```个主件和总容量```j```，讨论第```i```个主件的选法，只需要枚举以下这几种情况：
**a. 不选i主件；**
**b. 选i主件不选附件；**
**c. 选i主件+1附件；**
**d. 选i主件+2附件；**
**e. 选i主件+1、2附件。**
以上几种情况取最大值作为```dp[i][j]```。

于是写出了以下参考代码：
```cpp
#include<iostream>
using namespace std;

// v[i]、w[i]分别表示i物品的价格和重要度
// things2core[i]表示：若i是主件，则things2core[i]为该物品在core中第一维的索引;若i是附件，则为-1；
// core中第一维为主件维，第二维0位置存放主件自身index,1-2位置存放两个附件的index
int dp[32005],v[65],w[65],things2core[65],core[65][3];

int solve(int cores, int n){
    for(int i=1;i<=cores;i++){
        for(int j=n;j>=1;j--){
			
			int core_id = core[i][0];
			
            if(j>=v[core_id])
                dp[j] = max(dp[j], dp[j-v[core_id]]+v[core_id]*w[core_id]);
            if(j>=v[core_id]+v[core[i][1]] && core[i][1]!=0)
                dp[j] = max(dp[j], dp[j-v[core_id]-v[core[i][1]]] + v[core_id]*w[core_id] + v[core[i][1]]*w[core[i][1]]);
            if(j>=v[core_id]+v[core[i][2]] && core[i][2]!=0)
                dp[j] = max(dp[j], dp[j-v[core_id]-v[core[i][2]]] + v[core_id]*w[core_id] + v[core[i][2]]*w[core[i][2]]);
            if(j>=v[core_id]+v[core[i][1]]+v[core[i][2]] && core[i][1]*core[i][2]!=0)
                dp[j] = max(dp[j], dp[j-v[core_id]-v[core[i][1]]-v[core[i][2]]] + v[core_id]*w[core_id] + v[core[i][1]]*w[core[i][1]] + v[core[i][2]]*w[core[i][2]]);      
        }
    }
    return dp[n];
}

int main(){
    int n, m, coreid, core_num=0;

    cin >> n >> m; 
    
    for(int i=1;i<=m;i++) {
        cin>>v[i]>>w[i]>>coreid;
        if (coreid==0) {
            core[++core_num][0] = i;
            things2core[i] = core_num;
        }
        else{
        	if (core[things2core[coreid]][1]==0)
            	core[things2core[coreid]][1] = i;
            else core[things2core[coreid]][2] = i;
        }
    }

    cout << solve(core_num, n) << endl;

    return 0;
    
}

```

以上代码，可读性较差，太多的索引嵌套，可以将其封装成函数优化可读性；

阅读以上代码后，我们发现解决了dp转移方程的问题后，另一个难点是数据存储，正是因为我们想在数组中存储主件和附件的从属信息，导致代码中很多索引嵌套，晦涩难懂。此外，提交洛谷的测试点还会发现，Subtask #1中两个补充测试点WA，测试点信息如下：
