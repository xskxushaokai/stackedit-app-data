## T745016 公路乘车

### 题目描述

一个特别的单行街道在每公里处有一个汽车站。顾客根据他们乘坐汽车的公里数来付费。例如下表就是一个费用的单子。
没有一辆车子行驶超过 10 公里，一个顾客打算行驶 n 公里（1<=n<=100），它可以通过无限次的换车来完成旅程。最后要求费用最少。

### 输入格式

第一行十个整数分别表示行走 1 到 10 公里的费用（<=500）。注意这些数并无实际的经济意义，即行驶 10 公里费用可能比行驶一公里少。

第二行一个整数 n 表示，旅客的总路程数。

### 输出格式

仅一个整数表示最少费用。

### 输入输出样例 #1

#### 输入 #1

```
12 21 31 40 49 58 69 79 90 101
15
```

#### 输出 #1

```
147
```

#### 题目分析

定义状态```dp[i]```：行驶i公里的最小花费
`cost[j]` ：输入数据，表示连续乘坐 `j` 公里的费用

考虑顾客到达当前最后一站i时的前一站，共10种情况，即可以是从 ```i-1、i-2、i-3 ... i-10``` 中的任意一站来到第i站
因此只需要枚举10种情况，取最小即可

**状态转移方程为**：
$$dp[i]=min_{1 \le j \le 10}(dp[i−j]+cost[j])$$

**注意边界条件**：`dp[0] = 0`（起点费用为0）

#### 参考代码
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int cost[11];  // cost[i]表示坐i公里的费用
    for (int i = 1; i <= 10; i++) cin >> cost[i];
  
    int n;
    cin >> n;
  
    // 初值置为极大值
    // 数组方法为 memset(dp,0x3f,sizeof(dp));
    vector<int> dp(n + 1, INT_MAX); 
    dp[0] = 0;  // 起点
  
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= 10 && j <= i; j++) {
            // 从i-j公里处坐j公里到达i
            dp[i] = min(dp[i], dp[i - j] + cost[j]);
        }
    }
  
    cout << dp[n] << endl;
    return 0;
}
```

## T745110 抢金块

### 一、题目分析

#### 题意理解

- 地面上有 `n` 个格子，每个格子上有价值不等的金块

- 从第1个格子起跳，必须跳到最后一个格子（第n个）

- 每次可以跳 `S` 至 `T` 步（即步长范围是 `[S, T]`）

- 目标是获得最大总价值的金块


#### 关键信息

| 项目 | 说明 |
|:---|:---|
| 格子数 | `n < 1000` |
| 步长范围 | `2 <= S < T <= 10` |
| 金块价值 | `< 10000` |
| 起始点 | 第1个格子（直接获得其价值） |
| 终止点 | 第n个格子（必须到达） |


#### 样例解析

**输入：**
```
10
2 3
4 5 8 2 8 3 6 7 2 9
```

**最优路径：** 1 → 3 → 5 → 8 → 10
- 从1跳2步到3
- 从3跳2步到5
- 从5跳3步到8
- 从8跳2步到10

**总价值：** 4 + 8 + 8 + 7 + 9 = 36

  
### 二、DP思路分析

  

#### 状态定义

  

设 `dp[i]` 表示**跳到第 `i` 个格子时能获得的最大金块总价值**。

> 为什么选择"跳到第i个格子"作为状态？
> - 题目要求"最大化总价值"，是典型的最优化问题
> - 每个格子的选择依赖于前面格子的选择
> - 具有最优子结构：到达第i个格子的最优解，由到达前面某个格子的最优解转移而来

  
#### 状态转移方程

  

要到达第 `i` 个格子，可以从前面 `[i-T, i-S]` 范围内的某个格子 `j` 跳过来（步长在 `[S, T]` 之间）。

  

```
dp[i] = max(dp[j] + value[i])
// 其中 j ∈ [i-T, i-S] 且 j ≥ 1
```
**文字描述：** 到达第 `i` 个格子的最大价值 = 所有能一步跳到 `i` 的前驱格子 `j` 中，`dp[j]` 的最大值 + 当前格子的价值。

#### 边界条件
- `dp[1] = value[1]`：第一个格子的金块直接获得
- 其他 `dp[i]` 初始化为负无穷（表示暂时不可达）

#### 答案输出
`dp[n]`：到达最后一个格子的最大总价值

  

### 三、算法流程

```
初始化：
	dp[1] = value[1]
	dp[2..n] = -∞
遍历每个格子 i 从 2 到 n：
	遍历步长 step 从 S 到 T：
		j = i - step // 前驱格子
		如果 j >= 1：
			dp[i] = max(dp[i], dp[j] + value[i])
输出 dp[n]
```

### 四、参考代码

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;          // 格子个数
    int S, T;       // 步长范围 [S, T]
    
    // 输入
    cin >> n;
    cin >> S >> T;
    
    vector<int> value(n + 1);  // 每个格子的金块价值（1-based）
    for (int i = 1; i <= n; i++) {
        cin >> value[i];
    }
    
    // dp[i] 表示到达第i个格子的最大金块总价值
    // 初始化为负无穷，表示暂时不可达
    const int INF_NEG = -1e9;
    vector<int> dp(n + 1, INF_NEG);
    
    // 边界条件：第一个格子直接获得其价值
    dp[1] = value[1];
    
    // 状态转移
    for (int i = 2; i <= n; i++) {
        // 枚举所有可能的步长 [S, T]
        for (int step = S; step <= T; step++) {
            int j = i - step;  // 从第j个格子跳step步到达第i个格子
            
            // 确保前驱格子存在且可达
            if (j >= 1 && dp[j] != INF_NEG) {
                dp[i] = max(dp[i], dp[j] + value[i]);
            }
        }
    }
    
    // 输出答案：到达最后一个格子的最大总价值
    cout << dp[n] << endl;
    
    return 0;
}

```

### 五、复杂度分析

| 复杂度类型 | 数值 | 说明 |
|:---|:---|:---
| **时间复杂度** | O(n × (T-S+1)) | 遍历n个格子，每个格子枚举(T-S+1)个步长 |
| **空间复杂度** | O(n) | dp数组和value数组 |

  
由于 `n < 1000` 且 `T-S+1 <= 9`，实际运行非常快。


### 六、常见错误与注意事项

#### 错误1：忘记处理边界
```cpp
// 错误：没有判断 j >= 1
if (dp[j] != INF_NEG) // 当 j < 1 时会数组越界！
```

#### 错误2：初始化不当

```cpp
// 错误：初始化为0
dp[1] = 0; // 应该是 value[1]
```

#### 错误3：步长范围搞反

```cpp
// 错误：写成 step <= S 或 step >= T
for (int step = S; step <= T; step++) // 正确
```

## P1594 护卫队

### 一、题目分析

#### 1.1 题意理解

-   车队按固定顺序排队，不能超车
-   需要将车队分成若干组，每组同时过桥
-   每组总重量不能超过桥的最大承载量
-   每组通过时间 = 该组最慢车的过桥时间
-   求全部车队通过的最短总时间

#### 1.2 关键信息

项目|说明
---|---
桥最大载重|`maxW`
桥长度|`L`（千米）
车辆数|`n < 1000`
每辆车|重量 `W`，速度 `S`
通过时间|时间 = 距离/速度，单位转换为分钟

#### 1.3 过桥时间计算

一辆车通过桥的时间（分钟）：
```text
time = (L / S) * 60 分钟
```

### 二、DP思路分析

#### 2.1 状态定义

设 `dp[i]` 表示**前 `i` 辆车全部通过桥所需的最短时间**。

#### 2.2 状态转移方程

考虑最后一组车，假设最后一组包含从 `j` 到 `i` 的车（即第 `j, j+1, ..., i` 辆车为一组）。
```text
dp[i] = min(dp[j-1] + time(j, i))
其中 j ∈ [1, i] 且 weight(j,i) ≤ maxW
```
其中：
-   `weight(j, i)` = 第 `j` 到 `i` 辆车的总重量
-   `time(j, i)` = 第 `j` 到 `i` 辆车中速度最慢的那辆的过桥时间

#### 2.3 边界条件

-   `dp[0] = 0`（0辆车通过时间为0）

#### 2.4 答案输出

`dp[n]`，即前 `n` 辆车的最短通过时间

### 三、参考代码
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    double maxW;  // 桥最大载重（吨）
    double L;     // 桥长度（千米）
    int n;        // 车辆数
  
    cin >> maxW >> L >> n;
  
    vector<double> w(n + 1);  // 每辆车的重量
    vector<double> s(n + 1);  // 每辆车的速度
  
    for (int i = 1; i <= n; i++) {
        cin >> w[i] >> s[i];
    }
  
    // dp[i] 表示前i辆车全部通过的最短时间（分钟）
    // 初始化为无穷大
    const double INF = 1e18;
    vector<double> dp(n + 1, INF);
    dp[0] = 0;  // 边界条件：0辆车通过时间为0
  
    // 状态转移
    for (int i = 1; i <= n; i++) {
        double sumW = 0;      // 当前组的重量之和
        double minS = 1e18;   // 当前组的最慢速度
      
        // 枚举分组点j：最后一组包含[j, i]的车
        for (int j = i; j >= 1; j--) {
            sumW += w[j];           // 加入第j辆车
            minS = min(minS, s[j]); // 更新最慢速度
          
            // 如果超重，则不能再往前扩展了（前面的车更重，只会越加越重）
            if (sumW > maxW) break;
          
            // 计算[j, i]这一组的过桥时间（分钟）
            double groupTime = (L / minS) * 60.0;
          
            // 状态转移：前j-1辆车的时间 + 当前组的时间
            dp[i] = min(dp[i], dp[j - 1] + groupTime);
        }
    }
  
    // 输出结果，保留1位小数
    cout << fixed << setprecision(1) << dp[n] << endl;
  
    return 0;
}
```

#### 关键代码解析
```cpp
for (int j = i; j >= 1; j--) {
    sumW += w[j];
    minS = min(minS, s[j]);
  
    if (sumW > maxW) break;  // 剪枝：超重则停止
  
    double groupTime = (L / minS) * 60.0;
    dp[i] = min(dp[i], dp[j - 1] + groupTime);
}
```
-   **倒序枚举 `j`**：方便计算从 `j` 到 `i` 的累计重量和最慢速度
-   **剪枝优化**：一旦超重就 `break`，因为继续往前加只会更重

## P1434 滑雪

### 一、题目分析

#### 1.1 题意理解

- 给定一个 R×C 的高度矩阵
- 可以从任意点出发，滑向上下左右四个相邻点
- 只能滑向**高度严格减小**的相邻点
- 求最长滑坡的长度（经过的点数）

#### 1.2 关键信息
|项目|说明|
|---|---|
|矩阵大小|R × C，R,C ≤ 100|
|移动方向|上、下、左、右|
|移动条件|只能滑向高度更小的点|
|目标|最长路径长度（点数）|


### 二、解题思路

#### 2.1 为什么用记忆化搜索？

这道题如果用普通DP顺推方法，状态转移顺序不好确定（有多个方向，形成DAG而非线性结构）。

**关键观察**：从高点滑向低点，天然形成有向无环图（DAG），可以用**记忆化搜索**（DFS + 记忆化）解决。

### 2.2 状态定义

设 `dp[i][j]` 表示**从点 (i, j) 出发，能滑行的最长路径长度**。

### 2.3 状态转移方程
```shell
dp[i][j] = max(dp[ni][nj]) + 1

其中 (ni, nj) 是 (i,j) 的四个相邻点，且 height[ni][nj] < height[i][j]
```

如果没有可滑向的相邻点（四周都比当前高或相等），则 `dp[i][j] = 1`。

#### 2.4 记忆化搜索

- 对每个点进行DFS
- 如果 `dp[i][j]` 已经计算过，直接返回
- 否则递归计算四个方向的值，取最大值 + 1

### 三、完整参考代码
```cpp
#include <bits/stdc++.h>
using namespace std;

int R, C;
vector<vector<int>> h;      // 高度矩阵
vector<vector<int>> dp;     // 记忆化数组，dp[i][j]表示从(i,j)出发的最长路径
int dx[4] = {-1, 1, 0, 0};  // 上下左右四个方向
int dy[4] = {0, 0, -1, 1};

// 记忆化搜索：返回从(x,y)出发的最长滑坡长度
int dfs(int x, int y) {
    // 已经计算过，直接返回
    if (dp[x][y] != -1) {
        return dp[x][y];
    }
  
    dp[x][y] = 1;  // 至少包含自己这一个点
  
    // 枚举四个方向
    for (int i = 0; i < 4; i++) {
        int nx = x + dx[i];
        int ny = y + dy[i];
      
        // 检查边界
        if (nx < 0 || nx >= R || ny < 0 || ny >= C) continue;
      
        // 只能滑向高度更小的点
        if (h[nx][ny] < h[x][y]) {
            dp[x][y] = max(dp[x][y], dfs(nx, ny) + 1);
        }
    }
  
    return dp[x][y];
}

int main() {

    cin >> R >> C;

	// vector初始化的方法，R个元素，每个元素又是一个包含C个元素的vector，即 R行*C列的 二维矩阵。
    h.assign(R, vector<int>(C));
    dp.assign(R, vector<int>(C, -1));  // 初值-1，表示未计算
  
    for (int i = 0; i < R; i++) {
        for (int j = 0; j < C; j++) {
            cin >> h[i][j];
        }
    }
  
    int ans = 0;
  
    // 每个点都作为出发点找最长路径，最后取最长的路径
    for (int i = 0; i < R; i++) {
        for (int j = 0; j < C; j++) {
            ans = max(ans, dfs(i, j));
        }
    }
  
    cout << ans << endl;
  
    return 0;
}
```


### 六、顺推DP的写法：按高度排序后DP

按高度从小到大排序后，再从最低点开始顺推状态转移方程：
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int R, C;
    cin >> R >> C;
  
    vector<vector<int>> h(R, vector<int>(C));
    vector<vector<int>> dp(R, vector<int>(C, 1));  // 每个点至少为1
  
    // 存储所有点，按高度排序
    vector<tuple<int, int, int>> points;  // (高度, x, y)
  
    for (int i = 0; i < R; i++) {
        for (int j = 0; j < C; j++) {
            cin >> h[i][j];
            points.push_back({h[i][j], i, j});
        }
    }
  
    // 按高度从小到大排序
    sort(points.begin(), points.end());
  
    int dx[4] = {-1, 1, 0, 0};
    int dy[4] = {0, 0, -1, 1};
    int ans = 1;
  
    // 按高度从小到大处理（从低处往高处推）
    for (auto& [height, x, y] : points) {
        // 检查四个方向，看是否有更高的点可以转移过来
        for (int i = 0; i < 4; i++) {
            int nx = x + dx[i];
            int ny = y + dy[i];
            if (nx < 0 || nx >= R || ny < 0 || ny >= C) continue;
            // 如果相邻点更高，则那个点可以滑向当前点
            if (h[nx][ny] > h[x][y]) {
                dp[nx][ny] = max(dp[nx][ny], dp[x][y] + 1);
            }
        }
    }
  
    // 找最大值
    for (int i = 0; i < R; i++) {
        for (int j = 0; j < C; j++) {
            ans = max(ans, dp[i][j]);
        }
    }
  
    cout << ans << endl;
  
    return 0;
}
```

## P1115 最大子段和

### 题目描述

给出一个长度为 $n$ 的序列 $a$，选出其中连续且非空的一段使得这段和最大。

### 输入格式

第一行是一个整数，表示序列的长度 $n$。

第二行有 $n$ 个整数，第 $i$ 个整数表示序列的第 $i$ 个数字 $a_i$。

### 输出格式

输出一行一个整数表示答案。

### 输入输出样例 #1

#### 输入 #1

```
7
2 -4 3 -1 2 -4 3

```

#### 输出 #1

```
4
```

### 说明/提示

#### 样例 1 解释

选取 $[3, 5]$ 子段 $\{3, -1, 2\}$，其和为 $4$。

#### 数据规模与约定

- 对于 $40\%$ 的数据，保证 $n \leq 2 \times 10^3$。
- 对于 $100\%$ 的数据，保证 $1 \leq n \leq 2 \times 10^5$，$-10^4 \leq a_i \leq 10^4$。

---
### 题目分析

**核心问题**：找连续子数组，使和最大。

#### 状态定义
设 `dp[i]` 表示**以第 i 个元素结尾**的连续子数组的最大和。

> 为什么要"以i结尾"？因为这样可以保证连续性，方便状态转移。

#### 状态转移方程

对于以第 `i` 个元素结尾的子数组，有两种选择：

1.  **单独成段**：只选 `a[i]` 自己
2.  **接在前面**：把 `a[i]` 接到以 `i-1` 结尾的最优子段后面

$$dp[i]=max(a[i], dp[i−1]+a[i])$$

#### 最终答案
$$answer=max_{1≤i≤n}​{dp[i]}$$

### 参考代码（利用滚动变量优化空间）
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
  
    int n;
    cin >> n;
  
    long long ans = -1e18;      // 全局最大子段和
    long long dp = 0;           // 以当前位置结尾的最大子段和（滚动优化）
  
    for (int i = 1; i <= n; i++) {
        long long x;
        cin >> x;
      
        // 状态转移：要么接前面，要么重新开始
        dp = max(dp + x, x);
      
        // 更新全局答案
        ans = max(ans, dp);
    }
  
    cout << ans << endl;
    return 0;
}
```

## P1833 樱花

#### 题目分析

典型**混合背包**问题

三种树：最多看一遍、最多看$P_i$遍、可看无数遍
背包体积：总时间
物品数：n
体积：看树所需的时间$T_i$
价值：树的美学值$C_i$

#### 参考代码

```cpp
#include<iostream>
using namespace std;

int dp[1005];
int t[10005], c[10005], nums[10005];  // 体积，价值，数量

int solve(int n, int T){
    int maxp;
    for(int i=1;i<=n;i++){
        for(int j=T;j>=t[i];j--){
            if (nums[i]==0) maxp = j/t[i];
            else maxp = nums[i];
            for(int k=0;k<=maxp&&k*t[i]<=j;k++)
                dp[j] = max(dp[j], dp[j-k*t[i]]+k*c[i]);
        }
    }
    return dp[T];
}

int main(){
    int hs, ms, he, me, n;
    char ch;
    cin>>hs>>ch>>ms>>he>>ch>>me>>n;
    for(int i=1;i<=n;i++){
        cin>>t[i]>>c[i]>>nums[i];
    }
    cout << solve(n, (he-hs)*60+me-ms) << endl;
    return 0;
}
```

## P2347 砝码称重

### 一、题目分析

**问题本质**：这是一个**多重背包**的变形问题。给定6种砝码，每种砝码有数量限制，问能称出多少种不同的重量。

**关键特点**：

-   每种砝码最多用 `a[i]` 个（数量受限）
-   总重量上限为1000
-   不需要输出具体方案，只问"能否凑出"

----------

### 二、状态定义

```cpp
dp[i][j]  // 布尔值：前i种砝码能否凑出重量j
```

维度|含义
---|---
`i`|考虑前 `i` 种砝码（1~6）
`j`|目标重量（0~1000）
值|`1` 表示可以凑出，`0` 表示不能

----------

### 三、状态转移方程

对于第 `i` 种砝码（重量为 `w[i]`，数量为 `a[i]`），枚举使用 `k` 个：

```text
dp[i][j] = OR{ dp[i-1][j - k*w[i]] }  
其中 k ∈ [0, a[i]] 且 k*w[i] ≤ j
```

**解释**：要凑出重量 `j`，可以选择：

-   用0个第 `i` 种砝码 → 依赖 `dp[i-1][j]`
-   用1个第 `i` 种砝码 → 依赖 `dp[i-1][j-w[i]]`
-   ...
-   用 `k` 个第 `i` 种砝码 → 依赖 `dp[i-1][j-k*w[i]]`

只要其中任意一种情况可行，`dp[i][j]` 就为真。

----------

### 四、初始化与边界

```cpp
// 重量为0永远可以凑出（一个砝码都不用）
for(int i = 0; i <= N; i++) 
    dp[i][0] = 1;
```

**为什么 `dp[i][0] = 1`？**

-   凑出重量0的方法：一个砝码都不用
-   这是所有状态的起点

----------

### 五、完整代码详解

```cpp
#include<iostream>
using namespace std;

const int N = 6, J = 1000;
int dp[N+3][1005];      // dp[i][j]：前i种砝码能否凑出重量j
int a[N+3];             // a[i]：第i种砝码的数量
int w[] = {0, 1, 2, 3, 5, 10, 20};  // 砝码重量，w[0]占位

void solve() {
    // 三重循环：物品种类 × 目标重量 × 使用个数
    for(int i = 1; i <= N; i++)           // 枚举第i种砝码
        for(int j = 1; j <= J; j++)       // 枚举目标重量j
            for(int k = 0; k <= a[i] && k*w[i] <= j; k++)  // 枚举使用k个
                dp[i][j] = dp[i][j] || dp[i-1][j - k*w[i]];
}

int main() {
    // 读入每种砝码的数量
    for(int i = 1; i <= N; i++) 
        cin >> a[i];
  
    // 初始化：重量0总是可达
    for(int i = 0; i <= N; i++) 
        dp[i][0] = 1;
  
    solve();
  
    // 统计答案：统计dp[6][1]到dp[6][1000]中有多少个1
    int ans = 0;
    for(int j = 1; j <= 1000; j++) 
        ans += dp[N][j];
  
    cout << "Total=" << ans << endl;
    return 0;
}
```

----------

### 六、复杂度分析

维度|复杂度|说明
---|---|---
**时间**|O(N × J × max(a[i]))|6 × 1000 × 最多数量
**空间**|O(N × J)|约 6 × 1001 ≈ 6KB

实际运行完全在限制范围内。

----------

### 七、易错点总结

易错点|正确做法
---|---
忘记初始化 `dp[i][0]`|所有 `dp[i][0]` 都要设为1
统计答案时包含重量0|题目明确要求"不包括一个砝码也不用"，所以从 `j=1` 开始统计
`k` 的循环条件|既要 `k <= a[i]`，也要 `k*w[i] <= j`，防止数组越界
砝码重量数组|`w[0]` 是占位符，实际从 `w[1]` 开始用

----------

### 八、思路总结

```text
问题转化：多重背包 → 判断可行性
核心思想：逐种考虑砝码，记录所有可达重量
状态设计：dp[i][j] 表示前i种能否凑出j
转移方式：枚举当前砝码使用个数，或运算合并状态
```

这是一道经典的**动态规划入门题**，掌握它有助于理解更复杂的多重背包问题。

## T677613 暗黑破坏神
 

### 题目核心分析 
这是一道**完全背包变式题**，核心特征： 
1. 有 `n` 个魔法（物品），`m` 个金币（背包容量）； 
2. 每个魔法可升级多次（完全背包：物品可重复选）； 
3. 魔法不同等级对应**不同效果值**（价值随选择次数变化）； 
4. 题目要求：输出**最大效果值** 和 **每个魔法的等级**，多解时输出**花费金币最少**的方案。 

**所以我们分两步来完成，首先使用完全背包求出最大效果值，再想办法记录花费金币最少的方案**

--- 

### 步骤一：纯完全背包求解「最大效果值」 
#### 模型转化 
- 背包容量：金币总数 `m` 
- 物品：每个魔法（可重复升级） 
- 物品花费：魔法每级价格 `c[i]` 
- 物品价值：不同等级的效果值 `w[i][k]` 
- 目标：不超过容量，总价值（效果值）最大 

#### 状态定义
 `dp[j]`：花费 `j` 金币能获得的**最大效果值** 
 
#### 状态转移 
 逆序枚举金币（完全背包标准写法），枚举每个魔法的所有等级，更新最大效果： 
 ```cpp 
 dp[j] = max(dp[j], dp[j-k*c[i]]+w[i][k]); 
 ``` 
 
 #### 求「最大效果值」完整代码
 
```cpp 
# include<iostream>
# include<cstring>
using namespace std;

int dp[505];
// c[]-每种物品价格，p[]-记录每种物品拥有的等级数，w[][]-每个魔法不同等级拥有的价值
int c[105], p[105], w[105][55];

int solve(int n, int m){
    for(int i=1;i<=n;i++)
        for(int j=m;j>=1;j--)
            for(int k=0;k<=p[i]&&j>=k*c[i];k++){   // j>=k*c[i]的判断条件一定不能少
                dp[j] = max(dp[j], dp[j-k*c[i]]+w[i][k]);
            }
    return dp[m];
}

int main(){
    int n, m, t;
    cin >> n >> m;
    for(int i=1;i<=n;i++){
        cin >> c[i] >> p[i];
        for(int k=1;k<=p[i];k++)
            cin >> w[i][k];
    }
    cout << solve(n, m) << endl;
    return 0;
}

``` 

  #### 代码说明 
  - 只求解**最大效果值**，无路径记录、无最小花费判断； 
  - 适用于只需要答案数值，不需要具体方案的场景。 
  
  --- 
  
  ### 步骤二：增加「最小花费」+「路径回溯」输出完整方案 
  #### 升级需求 
  1. 多解时，选择**花费金币最少**的方案； 
  2. 记录每个魔法的等级，**回溯输出方案**。 
  
  #### 新增状态定义 
  1. `min_cost[j]`：获得 `dp[j]` 最大效果时，**实际花费的最小金币数**； 
  2. `num[i][j]`：处理第 `i` 个魔法、剩余 `j` 金币时，该魔法学习的等级（用于回溯）。 

#### 升级转移规则（两个关键字条件）
```cpp 
/* 
效果值更大时，直接使用对应方案
效果值相同时，比较花费
*/

// i物品装k个价值更大
if(dp[j-k*c[i]]+w[i][k]>dp[j]) {
    num[i][j] = k;
    min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
}
// i物品装k个的价值和上一次擂台的价值相同，则比较花费
else if(dp[j-k*c[i]]+w[i][k]==dp[j]){
    // min_cost[j] = min(min_cost[j-k*c[i]] + k*c[i], min_cost[j]);
    if(min_cost[j-k*c[i]] + k*c[i] < min_cost[j]) {
        num[i][j] = k;
        min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
    }
}


``` 
#### 路径回溯逻辑 
1. 从后往前遍历所有魔法； 
2. 通过 `num[i][j]` 拿到当前魔法的等级； 
3. 扣除对应花费，继续回溯上一个魔法； 
4. 最终输出所有魔法的等级。

 #### 完整代码
```cpp 
// 多重背包变体
// 每个技能是一种物品，可以对同一个技能连续升级，所以是多重背包
// 技能的不同等级拥有不同的效果值，所以可以看作是价值会变化的多重背包
// 因为每种物品的价值会随装的数量而变化，所以只能使用枚举每种物品的所有等级打擂台的方式
// 金币数量即为背包容量

# include<iostream>
# include<cstring>
using namespace std;

int dp[505];
// c[]-每种物品价格，p[]-记录每种物品拥有的等级数，w[][]-每个魔法不同等级拥有的价值
int c[105], p[105], w[105][55];

// 此外，本题还要求记录每个物品的选取方案、花费的金币，总价值相同时，使用花费金币最少的情况，贪心
int num[105][505];  // 记录第i个物品，在总空间为j时，应使用的数量
int min_cost[505];  // 对应dp数组，记录每个最大总价所需的最小花费

int solve(int n, int m){
    for(int i=1;i<=n;i++)
        for(int j=m;j>=1;j--)
            for(int k=0;k<=p[i]&&j>=k*c[i];k++){   // j>=k*c[i]的判断条件一定不能少

                // i物品装k个价值更大
                if(dp[j-k*c[i]]+w[i][k]>dp[j]) {
                    num[i][j] = k;
                    min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
                }
                // i物品装k个的价值和上一次擂台的价值相同，则比较花费
                else if(dp[j-k*c[i]]+w[i][k]==dp[j]){
                    // min_cost[j] = min(min_cost[j-k*c[i]] + k*c[i], min_cost[j]);
                    if(min_cost[j-k*c[i]] + k*c[i] < min_cost[j]) {
                        num[i][j] = k;
                        min_cost[j] = min_cost[j-k*c[i]] + k*c[i];
                    }
                }
                
                dp[j] = max(dp[j], dp[j-k*c[i]]+w[i][k]);
                
            }
    return dp[m];
}

int main(){
    int n, m, t;
    cin >> n >> m;
    for(int i=1;i<=n;i++){
        cin >> c[i] >> p[i];
        for(int k=1;k<=p[i];k++)
            cin >> w[i][k];
    }
    cout << solve(n, m) << endl;

    int res[105];  // 存储最终方案每个魔法的等级
    // 回溯找到最大价值下每种物品装入的数量
    int now = m;
    for (int i = n; i >= 1; i--) {
        res[i] = num[i][now];
        now -= res[i] * c[i];  // 减去当前魔法的花费
    }
    
    // 按顺序输出
    for (int i = 1; i <= n; i++) {
        cout << res[i] << endl;
    }

    return 0;
}

``` 

#### 代码说明 
1. **双关键字优化**：优先效果最大，相同效果选花费最少； 
2. **路径记录**：`num` 数组完整保存每一步选择，回溯即可得到方案； 
3. **边界处理**：数组初始化保证只有合法状态可以转移，无错误解。 

 --- 
 
### 总结 
1. **基础版**：纯完全背包，仅求解最大效果值； 
2. **进阶版**：增加最小花费判断 + 路径回溯，满足题目全部输出要求； 
3. 核心思想：**背包模板 + 双关键字DP + 路径回溯**，是背包问题输出方案的通用解法。

## P1064 [NOIP 2006 提高组] 金明的预算方案

> 此题不仅考察背包DP的应变能力，还考察代码实现基本功、问题分析能力

#### 整理题意：
总容量n，物品数m，物品体积为价格v[j]，最大化目标为v[j]*w[j]

**根据题目场景可得到的隐含信息**：每个物品最多只能买1个（0/1背包）。

**特殊点**：每个物品有主、附属性，一个主件最多有两个附件。

首先容易想到采用背包dp的思路，枚举前```i```个物品，枚举所有容量，计算每个状态时，枚举```i```物体的两种选择方式（1. 不装```i```，2. 装```i```同时连同对应的主件一起装），但很快会发现，将```i```物品连同主件一起装无法用我们通用的状态转移方程来表示，因为背包dp中状态```dp[i][j]```表示的是【前```i```个物品】,而这里```i```物品与他的主件并不是两个连续的物品，状态无法转移，所以行不通。**(可以尝试写一下)**

不妨换一个思路，**每个主件包含的附件数量是有限的**【0、1、2】，所以如果我们**对主件和背包容量建立状态```dp[i][j]```**，则状态可以连续转移，对于前```i```个主件和总容量```j```，讨论第```i```个主件的选法，只需要枚举以下这几种情况：
**a. 不选i主件；**
**b. 选i主件不选附件；**
**c. 选i主件+1附件；**
**d. 选i主件+2附件；**
**e. 选i主件+1、2附件。**
以上几种情况取最大值作为```dp[i][j]```。

于是写出了以下参考代码：
```cpp
#include<iostream>
using namespace std;

// v[i]、w[i]分别表示i物品的价格和重要度
// things2core[i]表示：若i是主件，则things2core[i]为该物品在core中第一维的索引;若i是附件，则为-1；
// core中第一维为主件维，第二维0位置存放主件自身index,1-2位置存放两个附件的index
int dp[32005],v[65],w[65],things2core[65],core[65][3];

int solve(int cores, int n){
    for(int i=1;i<=cores;i++){
        for(int j=n;j>=1;j--){
			
			int core_id = core[i][0];
			
            if(j>=v[core_id])
                dp[j] = max(dp[j], dp[j-v[core_id]]+v[core_id]*w[core_id]);
            if(j>=v[core_id]+v[core[i][1]] && core[i][1]!=0)
                dp[j] = max(dp[j], dp[j-v[core_id]-v[core[i][1]]] + v[core_id]*w[core_id] + v[core[i][1]]*w[core[i][1]]);
            if(j>=v[core_id]+v[core[i][2]] && core[i][2]!=0)
                dp[j] = max(dp[j], dp[j-v[core_id]-v[core[i][2]]] + v[core_id]*w[core_id] + v[core[i][2]]*w[core[i][2]]);
            if(j>=v[core_id]+v[core[i][1]]+v[core[i][2]] && core[i][1]*core[i][2]!=0)
                dp[j] = max(dp[j], dp[j-v[core_id]-v[core[i][1]]-v[core[i][2]]] + v[core_id]*w[core_id] + v[core[i][1]]*w[core[i][1]] + v[core[i][2]]*w[core[i][2]]);      
        }
    }
    return dp[n];
}

int main(){
    int n, m, coreid, core_num=0;

    cin >> n >> m; 
    
    for(int i=1;i<=m;i++) {
        cin>>v[i]>>w[i]>>coreid;
        if (coreid==0) {
            core[++core_num][0] = i;
            things2core[i] = core_num;
        }
        else{
        	if (core[things2core[coreid]][1]==0)
            	core[things2core[coreid]][1] = i;
            else core[things2core[coreid]][2] = i;
        }
    }

    cout << solve(core_num, n) << endl;

    return 0;
    
}

```

以上代码，可读性较差，太多的索引嵌套，可以将其封装成函数优化可读性；

阅读以上代码后，我们发现解决了dp转移方程的问题后，另一个难点是数据存储，正是因为我们想在数组中存储主件和附件的从属信息，导致代码中很多索引嵌套，晦涩难懂。此外，提交洛谷的测试点还会发现，Subtask #1中两个补充测试点**WA**，测试点信息如下：
```
输入： 
100 3
1000 5 3
10 5 3
50 2 0 

输出：
150
```

**仔细分析就会发现原因**：以上代码读入输入数据的方式仅适用于 先读入主件信息，再读入附件信息。
如果附件在其所属的主件之前输入，读取附件信息时， ```core[things2core[coreid]][1]```中```coreid```是主件索引，但此时```things2core[coreid]```为数组初值```0```，就会将附件信息存在```core[0]```中，而当读取主件信息时，```core[++core_num][0] = i```，将主件存在了```core[core_num]```中，导致主件和附件匹配错乱 。

代码修复和优化的任务留给读者来自行思考和解决。
