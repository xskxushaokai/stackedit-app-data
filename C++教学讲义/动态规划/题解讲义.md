

## P2347 砝码称重

### 一、题目分析

**问题本质**：这是一个**多重背包**的变形问题。给定6种砝码，每种砝码有数量限制，问能称出多少种不同的重量。

**关键特点**：

-   每种砝码最多用 `a[i]` 个（数量受限）
-   总重量上限为1000
-   不需要输出具体方案，只问"能否凑出"

----------

### 二、状态定义

```cpp
dp[i][j]  // 布尔值：前i种砝码能否凑出重量j
```

维度|含义
---|---
`i`|考虑前 `i` 种砝码（1~6）
`j`|目标重量（0~1000）
值|`1` 表示可以凑出，`0` 表示不能

----------

### 三、状态转移方程

对于第 `i` 种砝码（重量为 `w[i]`，数量为 `a[i]`），枚举使用 `k` 个：

```text
dp[i][j] = OR{ dp[i-1][j - k*w[i]] }  
其中 k ∈ [0, a[i]] 且 k*w[i] ≤ j
```

**解释**：要凑出重量 `j`，可以选择：

-   用0个第 `i` 种砝码 → 依赖 `dp[i-1][j]`
-   用1个第 `i` 种砝码 → 依赖 `dp[i-1][j-w[i]]`
-   ...
-   用 `k` 个第 `i` 种砝码 → 依赖 `dp[i-1][j-k*w[i]]`

只要其中任意一种情况可行，`dp[i][j]` 就为真。

----------

### 四、初始化与边界

```cpp
// 重量为0永远可以凑出（一个砝码都不用）
for(int i = 0; i <= N; i++) 
    dp[i][0] = 1;
```

**为什么 `dp[i][0] = 1`？**

-   凑出重量0的方法：一个砝码都不用
-   这是所有状态的起点

----------

### 五、完整代码详解

```cpp
#include<iostream>
using namespace std;

const int N = 6, J = 1000;
int dp[N+3][1005];      // dp[i][j]：前i种砝码能否凑出重量j
int a[N+3];             // a[i]：第i种砝码的数量
int w[] = {0, 1, 2, 3, 5, 10, 20};  // 砝码重量，w[0]占位

void solve() {
    // 三重循环：物品种类 × 目标重量 × 使用个数
    for(int i = 1; i <= N; i++)           // 枚举第i种砝码
        for(int j = 1; j <= J; j++)       // 枚举目标重量j
            for(int k = 0; k <= a[i] && k*w[i] <= j; k++)  // 枚举使用k个
                dp[i][j] = dp[i][j] || dp[i-1][j - k*w[i]];
}

int main() {
    // 读入每种砝码的数量
    for(int i = 1; i <= N; i++) 
        cin >> a[i];
  
    // 初始化：重量0总是可达
    for(int i = 0; i <= N; i++) 
        dp[i][0] = 1;
  
    solve();
  
    // 统计答案：统计dp[6][1]到dp[6][1000]中有多少个1
    int ans = 0;
    for(int j = 1; j <= 1000; j++) 
        ans += dp[N][j];
  
    cout << "Total=" << ans << endl;
    return 0;
}
```

----------

### 六、复杂度分析

维度|复杂度|说明
---|---|---
**时间**|O(N × J × max(a[i]))|6 × 1000 × 最多数量
**空间**|O(N × J)|约 6 × 1001 ≈ 6KB

实际运行完全在限制范围内。

----------

### 七、易错点总结

易错点|正确做法
---|---
忘记初始化 `dp[i][0]`|所有 `dp[i][0]` 都要设为1
统计答案时包含重量0|题目明确要求"不包括一个砝码也不用"，所以从 `j=1` 开始统计
`k` 的循环条件|既要 `k <= a[i]`，也要 `k*w[i] <= j`，防止数组越界
砝码重量数组|`w[0]` 是占位符，实际从 `w[1]` 开始用

----------

### 八、思路总结

```text
问题转化：多重背包 → 判断可行性
核心思想：逐种考虑砝码，记录所有可达重量
状态设计：dp[i][j] 表示前i种能否凑出j
转移方式：枚举当前砝码使用个数，或运算合并状态
```

这是一道经典的**动态规划入门题**，掌握它有助于理解更复杂的多重背包问题。

## T677613 暗黑破坏神
 

## 题目核心分析 
这是一道**完全背包变式题**，核心特征： 
1. 有 `n` 个魔法（物品），`m` 个金币（背包容量）； 
2. 每个魔法可升级多次（完全背包：物品可重复选）； 
3. 魔法不同等级对应**不同效果值**（价值随选择次数变化）； 
4. 要求：先求**最大效果值**，多解时输出**花费金币最少**的方案，最后输出每个魔法的等级。 
--- 
6. ## 步骤一：纯完全背包求解「最大效果值」 ### 1. 模型转化 - 背包容量：金币总数 `m` - 物品：每个魔法（可重复升级） - 物品花费：魔法每级价格 `c[i]` - 物品价值：不同等级的效果值 `w[i][k]` - 目标：不超过容量，总价值（效果值）最大 ### 2. 状态定义 `dp[j]`：花费 `j` 金币能获得的**最大效果值** ### 3. 状态转移 逆序枚举金币（完全背包标准写法），枚举每个魔法的所有等级，更新最大效果： ```cpp dp[j] = max(dp[j], dp[j - k*c[i]] + w[i][k]); ``` ### 4. 步骤一完整代码（仅求最大效果） ```cpp #include <iostream> #include <cstring> using namespace std; const int MAXM = 505; const int MAXN = 105; // dp[j]：花费j金币的最大效果值 int dp[MAXM]; // 魔法价格、最大等级、各等级效果 int c[MAXN], p[MAXN], w[MAXN][55]; int main() { int n, m; cin >> n >> m; // 初始化dp数组 memset(dp, 0, sizeof(dp)); // 输入数据 for (int i = 1; i <= n; i++) { cin >> c[i] >> p[i]; for (int k = 1; k <= p[i]; k++) { cin >> w[i][k]; } } // 完全背包核心：求最大效果值 for (int i = 1; i <= n; i++) { // 枚举每个魔法 for (int j = m; j >= 0; j--) { // 逆序枚举金币 // 枚举当前魔法的等级k for (int k = 0; k <= p[i] && j >= k * c[i]; k++) { dp[j] = max(dp[j], dp[j - k * c[i]] + w[i][k]); } } } // 输出最大效果值 cout << dp[m] << endl; return 0; } ``` ### 5. 代码说明 - 仅实现**最大效果值**求解，无路径记录、无最小花费判断； - 适用于只需要答案数值，不需要具体方案的场景。 --- ## 步骤二：增加「最小花费」+「路径回溯」输出完整方案 ### 1. 升级需求 1. 多解时，选择**花费金币最少**的方案； 2. 记录每个魔法的等级，**回溯输出方案**。 ### 2. 新增状态定义 1. `cost[j]`：获得 `dp[j]` 最大效果时，**实际花费的最小金币数**； 2. `path[i][j]`：处理第 `i` 个魔法、剩余 `j` 金币时，该魔法学习的等级（用于回溯）。 ### 3. 升级转移规则（双关键字判断） ```cpp // 效果更大 或 效果相同但花费更少 → 更新 if (new_val > dp[j] || (new_val == dp[j] && new_cost < cost[j])) ``` ### 4. 路径回溯逻辑 1. 从后往前遍历所有魔法； 2. 通过 `path[i][j]` 拿到当前魔法的等级； 3. 扣除对应花费，继续回溯上一个魔法； 4. 最终输出所有魔法的等级。 ### 5. 步骤二完整代码（AC最终版） ```cpp #include <iostream> #include <cstring> using namespace std; const int MAXN = 105; // 魔法最大数量 const int MAXM = 505; // 金币最大数量 // dp[j]：花费j金币的最大效果值 int dp[MAXM]; // cost[j]：最大效果下的最小花费 int cost[MAXM]; // path[i][j]：第i个魔法，j金币时，学习的等级 int path[MAXN][MAXM]; int c[MAXN]; // 每个魔法单级价格 int p[MAXN]; // 每个魔法最大等级 int w[MAXN][55];// 各等级效果值 int main() { int n, m; cin >> n >> m; // 初始化：dp设为极小值，cost设为极大值（保证最优解） memset(dp, -0x3f, sizeof(dp)); memset(cost, 0x3f, sizeof(cost)); dp[0] = 0; cost[0] = 0; // 输入数据 for (int i = 1; i <= n; i++) { cin >> c[i] >> p[i]; for (int k = 1; k <= p[i]; k++) { cin >> w[i][k]; } } // 核心DP：最大效果 + 最小花费 for (int i = 1; i <= n; i++) { for (int j = m; j >= 0; j--) { for (int k = 0; k <= p[i]; k++) { int use = k * c[i]; if (j < use) break; int new_val = dp[j - use] + w[i][k]; int new_cost = cost[j - use] + use; // 双关键字转移 if (new_val > dp[j] || (new_val == dp[j] && new_cost < cost[j])) { dp[j] = new_val; cost[j] = new_cost; path[i][j] = k; } } } } // 查找最优解：最大效果 + 最小花费 int max_val = 0, best_j = 0; for (int j = 0; j <= m; j++) { if (dp[j] > max_val || (dp[j] == max_val && cost[j] < cost[best_j])) { max_val = dp[j]; best_j = j; } } // 输出最大效果 cout << max_val << endl; // 回溯路径：输出每个魔法的等级 int res[MAXN]; int now = best_j; for (int i = n; i >= 1; i--) { res[i] = path[i][now]; now -= res[i] * c[i]; } // 按顺序输出方案 for (int i = 1; i <= n; i++) { cout << res[i] << endl; } return 0; } ``` ### 6. 代码核心亮点 1. **双关键字优化**：优先效果最大，相同效果选花费最少； 2. **路径记录**：`path` 数组完整保存每一步选择，回溯即可得到方案； 3. **边界处理**：数组初始化保证只有合法状态可以转移，无错误解。 --- ## 总结 1. **基础版**：纯完全背包，仅求解最大效果值，代码简洁； 2. **进阶版**：新增最小花费判断 + 路径回溯，满足题目全部输出要求； 3. 核心思想：**完全背包模板 + 双关键字DP + 路径回溯**，是背包问题输出方案的通用解法。
